ุสรุปสิ่งที่เรียนรู้
- ไม่ทำสิ่งนอกเหนือโปรแกรมที่ควรจะทำโดยไม่จำเป็น
- ใช้ style การ code ที่ใช้กันเป็นที่ยอมรับ เช่่น  ชื่อ class ขึ้นด้วยพิมพ์ใหญ่เเละต้องเป็น noun 
ชื่อ method มักจะเป็นชื่อการกระทำ/verb ขึ้นด้วยพิมพ์เล็ก
- การตั้งชื่อ class, method ให้  make sense กับโปรเเกรมที่จะทำ รวมทั้งชื่อทั้งสองนี้ต้อง make sense 
ไปทางที่สัมพันธ์กัน
- การตั้งชื่อ class, method, variable ต้องตั้งให้สื่อว่าเป็นอะไรหรือทำอะไร
- การตั้งชื่อ variable ควรตั้งชื่อเป็นในทางเดียวทั้งโปรแกรม เช่น noun ที่เป็นพหูพจน์จะเลือกเติม s หรือ ใช้เป็นคำที่เเสดงถึงเป็นสิ่งมากกว่า 1  เลือกมาหนึ่งอัน
- การ assign type variable ควรตั้งให้เหมาะสมกับ data ที่จะมาเก็บ 
- การสร้าง method เพื่อให้เรียกใช้  ควรสร้าง method ให้ผู้เรียกใช้ส่งเเค่ input มาพอจากนั้น method นี้เเค่ return output ออกให้ผู้เรียกใช้
 เสมือนว่าผู้ใช้รู้เเค่ว่าจะได้อะไรออกมาก็พอ โดยไม่จำเป็นต้องรู้การทำงาน
- method นั้นมีหน้าที่ตามชื่อของมันอยู่เเล้ว ถ้ามี logic อื่นเเยก เเม้จะนิดนึงควรเเยกเป็น method ใหม่
- การ return type ของ method ควรสัมพันธ์กับชื่อ method
- การเขียน code ควรให้กระชับ อะไรที่ลดรูปได้ควรลด เเต่ยังต้องให้ผู้อื่นอ่านรู้เรื่องด้วย
- ควรทำการ space, ย่อหน้า ให้ดีเพื่อความอ่านง่าย
